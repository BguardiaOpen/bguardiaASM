Tema 11 - Intercambio con otros sistemas

 

Objetivos

Conocer las técnicas prácticas para enviar y recibir información a otros sistemas integrados a Web.

  • Exportación de archivos
  • Importación de archivos
  • ODBC - bases de datos
  • Componentes Active X

 

Exportación de archivos

En diversos contextos es posible que nuestro sistema en Internet genere información que otros sistemas deben consumir. Por poner unos ejemplos simples:

  • Puede construirse un sistema de captura en Internet, que facilita que muchos usuarios capturen desde su máquina y se concentre la información en un solo servidor; si requiero procesar la información en otro sistema (por ejemplo, Excel) o en otra máquina y no el servidor de Internet, puedo pasar la información a través de archivos de texto.
  • Puede ser que el sistema en Internet haga cierto procesamiento pero parte tenga que hacerse por separado, entonces el sistema Internet podría pasarnos archivos con reportes de su operación o información sobre lo que hay que procesar. Por ejemplo, un sistema de comercio electrónico que haga autorizaciones manuales.
  • Un cliente puede requerir un reporte de su trabajo, por ejemplo un estado de cuenta o algún otro documento que se genere dinámicamente. El sistema Internet puede generar el archivo y luego proporcionarle la liga para bajarlo por FTP o similar.

De modo que una primera forma de intercambio de información entre nuestros sistemas Internet y otros sistemas, será que el sistema Internet genere archivos que puedan ser tomados por un usuario o un proceso automático, para ser usados en otras aplicaciones.

Entre las principales ventajas de este esquema, cabe hacer notar:

  • El sistema en Internet puede intercambiar información con sistemas que no cuentan con facilidades de comunicaciones.
  • Transferir la información capturada en Internet elimina los posibles errores en la transcripción.
  • El sistema en Internet puede generar la información en el formato soportado por las aplicaciones con las que cuenta el cliente u otras fases del proceso.

Como desventajas, hay que anotar:

  • Puede ser complejo establecer una esquema de seguridad para garantizar que los usuarios incorrectos no tengan acceso a los archivos exportados.
  • La construcción de un archivo complejo es lenta.
  • Se deben manejar riesgos de concurrencia.
  • Se requiere dar mantenimiento al servidor para que no se llene de archivos temporales.

Para implementar el proceso, utilizaremos el objeto Filesystem ("Scripting.FileSystemObject"), así como el objeto Archivo de texto ("TextStream"), con una sintaxis como la siguiente:

Set fso = CreateObject("Scripting.FileSystemObject")

Set Archivo = fso.CreateTextFile("c:\testfile.txt", True)

Archivo.WriteLine("This is a test.")

Archivo.Close

Resumiendo: se crea el archivo a través del File System, se escribe en él, y se proporciona una liga para accesarlo. Por ejemplo, un reporte de base de datos se puede presentar en un archivo .CSV (separado por comas), que luego podemos abrir en Excel:

 

Set fso = CreateObject("Scripting.FileSystemObject")

Set Archivo = fso.CreateTextFile("c:\InetPub\wwwroot\app\exporta.csv",True)

' Asumimos ya está el SQL

Set Reporte = Conn.Execute(SQL) ' ya está creada la conexión

do while not Reporte.EOF

' Supongamos que hay 8 campos

Texto=""

' Concatenar los valores, separados por comas...

for i=1 to 8

Texto=Texto & Reporte(i) & ","

next

' toda la linea que equivale a un registro

Archivo.WriteLine( Texto )

Reporte.MoveNext

loop

Archivo.Close

' Poner la liga

Response.Write "<a href=" & chr(34) & "exporta.csv" & chr(34) & ">"

Response.Write "Reporte en archivo</a>"

Importación de archivos

También nuestro sistema puede requerir de subir información a una base de datos; por ejemplo, si el usuario tiene un archivo de texto con la información generada por otro sistema, o si por cualquier otra razón (incluyendo seguridad) no podemos conectarnos directamente a la base de datos.

En este caso, hay que subir el archivo al servidor (por ejemplo, mediante un FTP o el servicio de Posting Acceptor) y luego correr el script ASP (o el objeto que corresponda) para que lo importe, es decir, lo procese.

Se usarán objetos muy similares a los anteriores, pero ahora leeremos el archivo y de acuerdo al archivo produciremos información. Inclusive puede servir esta idea para realizar consultas sencillas si no se tiene soporte a bases de datos.

En el ejemplo, se tiene un archivo CSV que se desea utilizar para localizar las calificaciones del alumno:

Const ForReading = 1, ForWriting = 2, ForAppending = 8

Set fso = CreateObject("Scripting.FileSystemObject")

Set Archivo = fso.OpenTextFile("c:\califs.csv", ForReading, True)

while not Archivo.AtEndOfStream

Linea = Archivo.ReadLine

if Left(Linea,6)= Matricula

Response.Write "Tus calificaciones: " & Linea

end if

loop

Archivo.Close

 

ODBC - Bases de datos

Cuando contamos con ODBC en el servidor, éste nos proporciona facilidades para acceder a diversos tipos de archivos o bases de datos, incluyendo archivos Excel, Access, DBase y similares. Por tanto, también se puede intercambiar información con otros sistemas, si podemos establecer una conexión ODBC con sus archivos o bases de datos.

El algoritmo general para importar/exportar información, en este caso, será:

  1. Abrir la conexión a la base de datos propia (de la aplicación Internet)
  2. Abrir la conexión al otro origen de datos (por ejemplo, conectarnos al archivo usado por la aplicación en el mainframe!!)
  3. Leer datos de una conexión
  4. Escribir datos en la otra conexión
  5. Ciclarse mientras haya datos
  6. Cerrar ambas conexiones

 

Componentes Active X

Finalmente, tenemos la opción de utilizar componentes ActiveX del lado del servidor. En este caso, programaremos en un lenguaje apropiado (Visual Basic, Delphi, Visual C++) un componente que cumpla la especificación COM Automation para proveer las funciones a ASP que hagan los procesos de importación o exportación; este componente podrá hacer uso de cualquier recurso del servidor, y por tanto convertir archivos al formato que se requiera o conectarse a donde se requiera para obtener la información.

El problema es que es más complejo de programar, y puede requerir intervención directa en el servidor. Por ello, será principalmente utilizada si tenemos control total del servidor y en Intranets. Sus ventajas incluyen la flexibilidad y la eficiencia.
