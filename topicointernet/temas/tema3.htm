Descripción de Java Script

Netscape, como parte de la evolución del estándar HTML, propuso utilizar un subconjunto de la sintaxis del lenguaje Java para controlar las propiedades del documento HTML; de este modo, creo un intérprete que permite ejecutar pequeños pedazos de código al tiempo que se está cargando una página HTML o se interactúa sobre ella. Originalmente llamado "LiveScript", por su parecido a Java (en la sintaxis solamente) finalmente se conoce como Java Script, y ha sido aceptado por distintos navegadores, si bien cada uno cuenta con distintas funciones de librería, por lo que es complejo lograr la compatibilidad entre los mismos.

De modo similar a Visual Basic versiones 4 y 5, es un lenguaje orientado a eventos con soporte a objetos ya definidos; sin embargo, no es orientado a objetos al no soportar dos de los tres principios básicos: herencia y polimorfismo (solamente soporta encapsulación).

Al ser orientado a eventos, generalmente el código Java Script se ejecutará en respuesta a eventos que suceden con la página, tales como el movimiento del ratón, presionar un botón en una forma, la carga de la página en el navegador, etcétera.

Para estudiar Java Script, necesitaremos examinar tres partes:

  • En qué contexto funciona.
  • La sintaxis.
  • El modelo de objetos.
  • Los eventos disponibles.
  • Cuándo es momento de usarlo.

 

Contexto de funcionamiento de Java Script

El concepto más importante y a veces confuso para manejar Java Script, es comprender que la ejecución de un script ocurre completamente en el lado del cliente; es decir, que el navegador hace las veces de intérprete y la ejecución se realiza en la computadora cliente, de modo que no se cuenta con acceso a los recursos del servidor, y solamente a un conjunto limitado de recursos del cliente.

Sintaxis

Java Script, como cualquier otro lenguaje de guiones (scripting) soportado por un navegador, se incluye dentro del código HTML dentro del tag SCRIPT; un ejemplo será el siguiente:

<script language="JavaScript">

document.write ("<H3>Hola mundo</H3>");

</script>

Un truco básico, importante para que no se muestre el código Java Script en navegadores que no soporten el lenguaje, es ponerlo entre comentarios HTML. De este modo, solamente un navegador que distinga scripts los tomará en cuenta. El ejemplo anterior quedaría de este modo:

<script language="JavaScript">

<!-- Inicia el comentario

document.write ("<H3>Hola mundo</H3>");

// aquí se pone doble slash para que esto sea un comentario de Java Script -->

</script>

Cabe aclarar que en un caso como el anterior, el código Java Script se ejecuta al momento en que se carga la parte de la página en que está contenido. Por lo que el resultado de este código se insertará después del HTML que precede al script, y antes del HTML que le sigue.

La sintaxis de instrucciones de Java Script es muy similar a la de C o C++; se basa en llamar funciones, las cuales van seguidas de paréntesis y en caso de tener parámetros, estos se colocan entre los paréntesis; y se separa caa instrucción por puntos y comas. Dado que es un lenguaje basado en objetos, muchas veces las funciones pertenecen a un objeto, por lo cual hacemos referencia al objeto y después, con un punto como separación, a la función o método dentro del objeto. En el ejemplo anterior, estamos usando del objeto document, que naturalmente se refiere al documento HTML que se está bajando, el método "write", cuya función es escribir una cadena en el documento.

Lo interesante de Java Script es que no tiene que ser una cadena fija, como en el ejemplo anterior, sino que puede ser variable y generada por un algoritmo en el programa. Antes de ver un ejemplo, mencionaremos algunos métodos más:

  • document.writeln (mensaje) - Genera una salida al documento, se le pasa como parámetro una cadena; la diferencia que tiene con document.write es que genera un salto de línea al final.
  • alert  (mensaje)  - Genera un cuadro de diálogo para confirmación, separado de la página HTML. Por defecto, solamente muestra un botón de OK.
  • prompt (mensaje, valordefecto)  - Genera un cuadro de diálogo para captura, separado de la página HTML. Por defecto, muestra el mensaje, un espacio para escribir (que muestra al inicio el valor por defecto), un botón de OK y uno de Cancel.
  • var Identificador[=valorinicial o expresión] - Permite declarar la existencia de una variable; el tipo se detecta en forma automática de acuerdo al contexto. Opcionalmente, puede inicializarse, dándole una cadena o un número como valor, o bien a través de una función, como en el siguiente ejemplo..
  • // y /* ... */ - al igual que en C++, permiten colocar comentarios dentro del código Java Script.
  • Al igual que en C y C++, se pueden agrupar los comandos en bloques mediante el uso de los corchetes { }
  • El lenguaje Java Script es sensitivo a mayúsculas y minúsculas, al igual que C o C++; por ejemplo, es diferente el significado de document.writeln y Document.WriteLn

El siguiente ejemplo puedes checarlo aquí.

<html>

<head><title>Ejemplo de Java Script</title>

</head>

<script language="JavaScript">

<!-- Inicia el comentario para que no se vea el código en navegadores que no soporten Java Script.

var Nombre=prompt("Bienvenido a nuestra página. ¿Cómo te llamas?", "");

// aquí se pone doble slash para que esto sea un comentario de Java Script -->

</script>

<body background="#FFFFF">

<script language="JavaScript">

<!-- Inicia el comentario

document.write ("<H3>¡Hola, ");

document.write (Nombre);

document.write ("!</H3>");

// aquí se pone doble slash para que esto sea un comentario de Java Script -->

</script>

</body></html>

 

Operadores

JavaScript permite utilizar casi todos los operadores que se soportan en C y C++, con el mismo significado, incluyendo los siguientes:

= += -= *= /=
%= + - * /
% ++ -- && ||
! == != > <
>= <= ? ... : ( )

Entre los operadores con un significado adicional, se encuentra la capacidad de manejar cadenas utilizando el signo de + como función de concatenación, tal como ocurre en Basic. La precedencia de los operadores es similar a C.

Para operaciones avanzadas, se tiene el objeto Math, que cuenta con distintos métodos correspondientes a funciones mátemáticas,incluyendo:

  • abs, acos, asin, atan, ceil, cos, exp, floor, log, max, min, pow, random, round, sin, sqrt y tan.

Funciones

Al igual que en C y C++, es posible (y recomendable) declarar funciones que agrupan una serie de instrucciones bajo una sola llamada, pueden recibir parámetros y regresar un resultado. Esto es especialmente conveniente si existen un proceso que deberá ejecutarse sucesivamente. Un ejemplo:

<script language="JavaScript">

<!-- Inicia el comentario para que no se vea el código en navegadores que no soporten Java Script.

function cubo(numero) {

    var cubo= numero * numero * numero;

return cubo;

}

Una importante ventaja de las funciones es que no se ejecutan en el momento de cargarse, sino hasta que se carga el código que las llama. Por ello, se recomienda que toda función se defina en el encabezado, para que se cargue antes del resto de la página y así cuando se desee usarla ya esté preparada.

Un ejemplo avanzado de funciones es la construcción de un calendario de acuerdo a la hora del día; una versión que está disponible en el Web puede verse en http://www.best.com/~nesssus/js-today.htm.

Modelo de objetos

Los objetos principales en JavaScript pueden dividirse en dos: aquellos de uso general, y aquellos que son proporcionados por el navegador.

Los objetos de uso general más importantes con los que contamos son:

Array

String

Math

Date

Los objetos que permiten el manejo del navegador, incluyen la siguiente jerarquía:

window

location

history

document

forms

anchors

Ya se han indicado los métodos principales del objeto Math, y hemos trabajado con el objeto document. Se mencionan a continuación algunos de los principales métodos de otros objetos:

Asociación de eventos a funciones

El llamado a funciones de Java Script no solamente se puede hacer al cargar código Java Script dentro de la página; también pueden conectarse a eventos en la página.

Para darnos una idea del funcionamiento de este concepto, los principales eventos disponibles en una página son:

Evento Tag al que está asociado Significado
onLoad BODY Cuando se termina de cargar la página y sus imágenes u otros archivos asociados
onUnload BODY Cuando se navega a otra página, saliendo de la misma.
onBlur INPUT (TODOS), SELECT, TEXTAREA Cuando el elemento correspondiente pierde el foco (por ejemplo, cuando se captura algo en un campo y posteriormente se cambia de campo)
onChange INPUT (TODOS), SELECT, TEXTAREA Cuando se cambia el valor del elemento correspondiente. Por ejemplo, al capturar.
onSubmit FORM Cuando se aprieta un botón SUBMIT (enviar) de una forma. Es el más usado para validar el contenido de la forma.
onFocus INPUT (TODOS), SELECT, TEXTAREA Cuando el elemento correspondiente toma el foco (por ejemplo, cuando se captura algo en otro campo y posteriormente se cambia al campo correspondiente)
onMouseOver A, IMG Cuando el ratón se coloca sobre la liga, aún sin apretar el botón.
onSelect INPUT (TEXT,PASSWORD), TEXTAREA Cuando se selecciona texto dentro del campo.
onClick INPUT (RADIO, CHECKBOX, BUTTON, SUBMIT, RESET) Cuando se presiona el botón del ratón sobre el campo (o se teclea espacio cuando tiene foco el campo)

 

Para relacionar un evento a una función, se realiza con la sintaxis siguiente:

<TAG onevento="funcion">

Por ejemplo, para llamar a la función Saludar (que debería definirse previamente, en el encabezado de la página) cuando se termina de cargar una página, el tag correspondiente sería:

<BODY onload="Saludar();">

En el ejemplo anexo, se tiene una página que contiene una forma, la cual es validada utilizando los eventos anteriores.

 

Cuándo usarlo

Los principales usos "clásicos" de Java Script incluyen:

  • Validación de formas.
  • Efectos visuales, principalmente como respuesta a eventos del ratón.
  • Control de applets Java (tema futuro) y objetos ActiveX (otro tema futuro).

En general, podemos usarlo para enriquecer la interfase con el usuario del lado del cliente, descargando parte del procesamiento que haría el servidor. De cualquier modo, es altamente recomendable siempre considerar pruebas en distintos navegadores y aún en distintas versiones del mismo navegador, tanto como con los scripts deshabilitados.
