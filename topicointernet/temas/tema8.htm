Modelo de objetos ActiveX Server Components

En clases pasadas, vimos los 5 objetos intrínsecos de ASP, que forman parte propiamente de la interfase de programación con el Servidor Web. Estos objetos representan directamente una parte del ISAPI, es decir, modelan las capacidades de programación del servidor Web.

Es posible contar con otros recursos en el servidor Web, que no forman parte propiamente dicha del mismo sino que son aplicaciones instaladas en la misma computadora; si estas aplicaciones siguen el estándar COM para exponer sus objetos a un lenguaje de programación, pueden ser programadas desde un script ASP.

Esto quiere decir que se pueden desarrollar aplicaciones, formadas por clases COM, y será posible utilizar estas clases desde cualquier cliente COM, por ejemplo una aplicación desarrollada en Visual Basic, Visual C++, o inclusive ASP.

Más adelante en el curso veremos como desarrollar componentes COM en otros lenguajes para ser usados en ASP (o en otros ambientes); en esta clase, se examinará cómo usar en ASP componentes ya desarrollados, y se utilizarán como ejemplos algunos que ya trae el IIS.

 

Creación de objetos ActiveX Server

Para crear un objeto ActiveX del lado del servidor, el script ASP cuenta con dos opciones:

  • Declarar el objeto y su alcance en el global.asa
  • Utilizar el método Server.CreateObject

En la próxima clase veremos el primer caso para crear objetos de acceso a datos; por el momento, utilizaremos solamente Server.CreateObject.

La sintaxis es:

Set Variable=Server.CreateObject(NombreClase)

Donde NombreClase es una cadena que permite referenciar en forma única al componente que se utilizará, según fue registrado en la base de datos interna de Windows.

Nótese que se usa Set, pues la variable será una referencia al objeto.

 

Ejemplo de objetos: Browser Capabilities

Este objeto está diseñado para analizar la información que envía el cliente, y en base a ello averigüar las capacidades con las que cuenta este cliente: si soporta JavaScript, Java, controles ActiveX, etcétera.

Para crear el objeto, utilizaremos Server.CreateObject con el nombre de clase "MSWC.BrowserType":

Set BrowserType = Server.CreateObject("MSWC.BrowserType")

Otros objetos disponibles

Si bien no vamos a analizarlos en este tema, se mencionan otros objetos que provee IIS:

  • AdvertisementRotator - Permite mostrar anuncios (banners) en una secuencia al azar o programada.
  • ADO - Proporciona acceso a bases de datos (semana próxima)
  • ContentLinking - Permite construir dinámicamente las ligas de un temario (por ejemplo, un libro); de modo que cada página automáticamente se liga a las anteriores y siguientes.
  • Dictionary - Permite crear pares llave/valor y hacer búsquedas sobre ellos.
  • Err - Proporciona manejo de códigos de error. Se usará en el acceso a bases de datos.
  • FileSystemObject - Permite acceder al sistema de archivos del servidor.
  • TextStream - Permite leer y escribir archivos de texto, asociado con FileSystemObject.
