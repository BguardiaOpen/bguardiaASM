Características generales de Java (Repaso)

Java es un lenguaje con mucha base en C y C++, pero totalmente orientado a objetos. A diferencia de C++, todo en Java es un objeto. Esto hace el lenguaje más sencillo, en particular, el uso de apuntadores se abstrae lo que simplifica hasta cierto punto la programación. Sin embargo, no es un lenguaje trivial para programar, es más complejo que Basic.

Otra característica importante de la mayoría de las implementaciones y la definición del estándar de Java, es que no es ni interpretado ni compilado, sino trabaja en un esquema intermedio llamado "máquina virtual". En este esquema, el código Java se compila a un código máquina definido específicamente para adaptarse a Java, pero que no es igual al código máquina del procesador donde se ejecutará; por lo que se requiere un software adicional, llamado "máquina virtual", el cual traduce del código máquina virtual Java al código máquina nativo de la computadora donde se ejecute.

Respecto a los lenguajes compilados a código nativo, esto da como ventaja la portabilidad a nivel binario y no sólo de código fuente, es decir, si creo una aplicación Java en la máquina virtual en una PC con Windows 95, se ejecutará igualmente en Sun, Macintosh, o cualquier otra plataforma donde se soporte la Java VM.

Respecto a los lenguajes interpretados, Java resulta más eficiente, se ejecuta más rápido al no requerir los procesos de análisis sintáctico y gramatical, sino solamente la traducción directa de los códigos máquina Java a máquina nativa.

Como contraparte, requiere una compilación compleja y es más lento que código nativo.

Applets

Concepto:

Son módulos o clases escritas en Java, restringidos a ejecutarse dentro de un navegador.

Características:

  • Muy limitados: no tienen acceso a archivos, muchas veces no pueden comunicarse a otro servidor salvo del que se bajaron, sólo pueden presentar información dentro de la ventana especificada en el navegador.
  • Muy seguros: difícilmente pueden causar daños a la máquina cliente o acceder a información indebida, si bien los errores en la implementación de las VM pueden permitir algunos accesos.
  • Se &ldquo;bajan&rdquo; del servidor, en forma inteligente (sólo si han cambiado desde la última vez que se accesaron), usando el protocolo estándar de HTTP.

Aplicaciones:

Los applets de Java pueden usarse para ejecutar código complejo dentro de un navegador; tienen acceso a diversos recursos, sin embargo limitados por seguridad. Entre los usos comunes de los applets podemos encontrar:

  • Gráficas
  • Interfases complejas
  • Envío y recepción de información compleja: encripción, análisis o conversión de la información.
  • Sistemas completos que viven en un navegador, ej. juegos.

Ventajas:

  • Son más rápidos y compactos que los scripts del lado del cliente.
  • El código no es directamente visible en el navegador, si bien existen herramientas para "descompilar" Java.
  • Son más seguros que los ActiveX, al no poder tomar cualquier acción arbitraria sobre la máquina del cliente.

 

Desventajas:

  • Pueden requerir bastante tiempo para bajarse, sobre todo la primera vez.
  • Son más difíciles de depurar.
  • Se requiere un servidor Web para probarlos, a diferencia de HTML y scripts que se pueden probar trabajando sobre archivos locales.
  • La programación Java es compleja, sin llegar al nivel de C; sin embargo, tiene bastantes trucos. Un diseñador aprende JavaScript o VBScript con facilidad, mientras que Java le resulta complejo; simplemente, son herramientas más sencillas y por tanto, más productivas en términos del tiempo de programación.
  • No son tan poderosos como los controles ActiveX, resultan limitados en comparación con ellos.

Servlets

Concepto:

Son módulos o clases escritas en Java, restringidos a ejecutarse en el contexto del servidor Web.

Aplicaciones:

Al igual que ASP, son una tecnología diseñada para producir documentos visibles en un navegador (primordialmente HTML y Java Script), utilizando los recursos del servidor en respuesta a solicitudes de HTTP.

Ventajas:

  • Es una tecnología soportada en diversas plataformas de servidores Web, estándar e independiente del fabricante
  • Es más rápida en su ejecución en relación a tecnologías interpretadas como ASP o Cold Fusion
  • El código está totalmente oculto del cliente, a diferencia de las tecnologías de scripts clientes o HTML.

Desventajas:

  • No es tan rápida como tecnologías nativas, tales como ISAPI, NSAPI ; por ello, también es menor su escalabilidad para dar soporte a grandes cantidades de usuarios concurrentes.
  • El soporte universal le proporciona funcionalidad limitada con respecto a explotar al máximo los recursos del sistema operativo o plataforma; cuando se aumenta una plataforma de servlets con las clases nativas que lo permiten, se pierde a cambio portabilidad.
  • La programación Java es compleja, sin llegar al nivel de C; sin embargo, tiene bastantes trucos. Por ello, es más productivo (en términos de tiempo para el desarrollador) usar lenguajes de Scripts en el lado del servidor como ASP.
