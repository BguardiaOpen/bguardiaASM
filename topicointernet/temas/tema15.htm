Style Sheets

Justificación:

Cuando se quiere aplicar un formato uniforme a un sitio Web, si ya es mayor de unas cuantas páginas, es bastante tardado modificar el diseño y adaptarlo a algo nuevo, utilizando HTML puro y tradicional (versión 3 y anteriores...)

Por otro lado, muchas veces deseamos darle formato a elementos tales como la información en una tabla; y se vuelve exageradamente tedioso cuando hay muchos elementos.

Por ejemplo, para conseguir la siguiente tabla:

Dato 1 Dato 2
Dato 3 Dato 4

El código que se necesita es:

<table border="1" cellpadding="0" cellspacing="0">

<tr>

<td><font face="Arial" size="1" color="#FF0000">Dato 1</font></td>

<td><font face="Arial" size="1" color="#FF0000">Dato 2</font></td>

</tr>

<tr>

<td><font face="Arial" size="1" color="#FF0000">Dato 3</font></td>

<td><font face="Arial" size="1" color="#FF0000">Dato 4</font></td>

</tr>

</table>

Nótese la repetición en el tag de Font que le está aplicando las pripiedades a cada una de las celdas.

Si todas las celdas de una página tienen el mismo estilo, y hay muchas (como en una tabla de calificaciones), se vuelve tedioso además de ineficiente.

Concepto:

Las hojas de estilo (Style Sheets) están basadas en un tag especial y nuevo a HTML 4.0 (<style>), el cual contendrá información sobre el formato que se desea aplicar para determinados Tags posteriores.

Puede incluirse una definición de estilo como parte del HTML en sí mismo:

        <style type="text/css"><!--
			.byline { margin-top: 5px }
			.subheadline { margin-bottom: 6px }
			.textfield { font-weight: bold; border: solid 1px #000000 }
			a { color: #cc0000; text-decoration: none }
			a:active { color: #cc3300 }
			a:visited { color: #cc0000 }
			font.reg10 { font-size: 10px; font-family: verdana, arial, helvetica }
			font.reg12 { font-size: 12px; font-family: verdana, arial, helvetica }
			font.reg9 { font-size: 9px; font-family: verdana, arial, helvetica }
			input { font-size: 10px; font-family: verdana, arial, helvetica }
			select { font-size: 10px; font-family: verdana, arial, helvetica }
			table { font-family: verdana, arial, helvetica }
			textarea { font-size: 10px; font-family: verdana, arial, helvetica; border: solid 1px #000000 }-->
	</style>

Por ejemplo, esta definición de estilo controla la tipografía y tamaños de ligas, fuente regular, campos de entrada y selección, así como un textarea.

Por otro lado, define el color de los links lo que permite controlar que no se use el color azul que por defecto tiene un link, ni el rojo cuando ya está visitada. En el ejemplo, el link es un rojo intenso que baja de intensidad al pasar el ratón sobre la liga (hover) y queda rojo aún después de visitar la liga (no cambia de color como una liga regular).

Pero cuando se desea difundirla en todo un sitio, es mucho más sencillo hacer una referencia a un archivo que contenga la definición de estilo (al que generalmente se denomina con la extensión .css)

Aplicaciones:

Los paquetes actuales de diseño de páginas generalmente ya permiten el uso de Style Sheets en forma semiautomática, lo que facilita la aplicación de un diseño, tipografía, colores y estilos de texto similares para todo un sitio Web. 

Ventajas:

Permiten realizar efectos visuales que normalmente no se podrían hacer sin ellas; además de simplificar la aplicación de un formato uniforme a una página o inclusive un sitio Web.

Desventajas:

Solamente son soportadas por Netscape 4.0 en adelante y Explorer 4.0 en adelante (3.0 parcialmente). De modo que si un porcentaje importante de los usuarios no cuentan con navegadores modernos, no disfrutarán de los beneficios.

Sin embargo, por la forma en que se definen los Tags en HTML el incluir un style sheet no afecta a los usuarios "tradicionales"; simplemente hay que cuidar que nuestro sitio sea visible aún sin contar con los Style Sheets.
