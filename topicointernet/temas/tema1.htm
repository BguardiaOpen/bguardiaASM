Tema 1 - Funcionamiento del Protocolo HTTP

Historia

Internet nace en los sesentas, como un proyecto de la agencia de investigaciones avanzadas del departamento de la Defensa de los Estados Unidos (ARPA).

Originalmente, solo existen unos cuantos nodos, correspondientes a agencias del gobierno de los EU y a las principales universidades; todos ellos son computadoras de buen tamaño, con acceso compartido en el tiempo de las terminales tontas. El sistema operativo UNIX aún estaba en sus principios, de modo que en primera instancia hubo una amplia diversidad de sistemas conectados a la ARPANET, como se conocía en estos tiempos.

La  característica que permitió unir a las computadoras en red, fue la estandarización en los protocolos de comunicación empleados; en una forma bastante democrática, a base de comités en línea y publicación de documentos "para comentarios" (RFCs=Request For Comments), es decir, para ser evaluados y perfeccionados por la comunidad investigadora. Mientras una computadora cumpliera los puntos estipulados en cierto número de RFCs, podía comunicarse con otra.

La familia de protocolos que nació en esta etapa, se conoce como TCP/IP, llamada así por sus dos principales protocolos: IP (Internet Protocol), y TCP (Transport Control Protocol).

Una vez que se determinó el estándar de los protocolos básicos en TCP/IP (que sigue evolucionando), los que permitieron la comunicación en redes complejas y entre sistemas heterogéneos, se fueron diseñando protocolos de nivel superior, que requieren TCP/IP como base. Estos "protocolos de aplicación" se desarrollaron pensando en necesidades específicas para el trabajo a través de computadoras comunicadas entre sí. Los dos primeros, de gran importancia hasta nuestros días, fueron telnet y ftp.

  • Telnet permite que una computadora se conecte a otra emulando una terminal de la segunda; esto permite que un usuario trabaje en una computadora remota, tal como lo haría en una terminal local de la misma, generalmente a través de una interfaz de comandos.
  • FTP (File Transfer Protocol) permite conectarnos a una computadora remota y transferir archivos con la computadora local. Permite un envío y recepción confiable de dichos archivos, además de funciones de administración tales como listado, creación y eliminación de directorios.

Posteriormente, con la difusión de sistemas UNIX y especialmente del correo interno a los mismos, se definió el protocolo SMTP (Simple Mail Transfer Protocol), el que permitió el nacimiento del correo Internet, en el cual un usuario puede enviar información a otro en forma de mensajes, sin importar que esté en otra sección de la Internet.

El crecimiento de Internet fue principalmente académico y relativamente lento, hasta 1992, cuando Tim Berners-Lee fusionó dos viejos conceptos: Internet y el hipertexto, creando el WWW (World Wide Web), que no es sino el conjunto de computadoras en Internet que soportan el protocolo creado por él: HTTP (Hyper Text Transfer Protocol). Este protocolo permite la visualización de documentos localizados en un servidor remoto, junto con las características de control y navegación que lo convierten en un "hiperdocumento", es decir, un documento que cuenta con ligas para explorar documentos relacionados.

Protocolo HTTP

HTTP: Hyper Text Transfer Protocol

Es el protocolo que se utiliza el cliente (browser) para comunicarse con el servidor (servidor Web) para la transferencia de información (principalmente, páginas Web).

Es un protocolo no orientado a conexión.

Esto significa que se envían los datos sin esperar recibir un mensaje de ‘Acknowledge’ (o confirmación que los datos se recibieron correctamente)

Solicitud de páginas al servidor

Cliente: Se establece el enlace con el servidor indicándose la página que se quiere obtener, mediante el URL

URL (Uniform Resource Locator): es la convención que permite identificar en forma única a cualquier recurso conectado a Internet.

Se envían las cabeceras de solicitud de la página web solicitadas

Servidor: Decodifica y valida la solicitud

Localiza la página Web solicitada

Genera las cabeceras de respuesta y devuelve la página solicitada al cliente

Cliente:

Recibe la página que el Servidor envió como respuesta.

Muestra la página Web solicitada

 

Cabeceras HTTP

Además del contenido mismo de la información que el servidor regresa al cliente, se requiere información para control de la comunicación. Esto se hace a través de "cabeceras", que el servidor envía al cliente antes del resto de la información.

Existen 5 tipos de cabeceras:

Códigos de estado informativo

Códigos de operación satisfactoria

Códigos de estado de redirección

Códigos de error del cliente

Códigos de error del servidor

 

Códigos de estado informativo

Son para funciones experimentales y sólo ofrecen información.

No se usan, pero están reservadas para uso futuro

Comienzan a partir del número 100

 

Códigos de operación Satisfatoria

La acción se ha recibido y aceptado satisfactoriamente

201 (Created)

La petición se ha llevado a cabo y ha dado como resultado, la creación de un nuevo recurso, al que puede referirse mediante el URL

202 (Accepted)

La solicitud fue aceptada para ser procesada, pero no se ha completado

204 (No Content)

El servidor ha llevado a cabo la solicitud, pero no hay información qué devolver.

 

Códigos de Redirección

Debe realizarse una acción adicional para llevar a cabo la solicitud

300 (Multiple Choices)

El recurso se encuentra disponible en varias ubicaciones y no es posible determinar la ubicación preferente

301 (Moved Permanently)

Se ha asignado un URL permanente y cualquier referencia futura se deberá hacer a las referencias del URL devuelto

302 (Moved Temporarily)

El recurso reside temporalmente en otro URL

 

Códigos de Error del Cliente

La solicitud, contiene una sintaxis errónea o no puede llevarse a cabo

400 (Bad Request)

No se ha llevado a cabo la solicitud, debido a un error de sintaxis

401 (Unauthorized)

Se solicita una clave de acceso, la cual el cliente no ha podido ofrecer.

403 (Forbidden)

Se ha comprendido la solicitud, pero el servidor se niega a devolver una respuesta, por algún motivo

404 (Not Found)

No se ha encontrado un URL que coincida con la petición.

 

Códigos de Error del Servidor

El servidor, ha fracasado en su intento de llevar una solicitud, aparentemente válida.

500 (Internal Server Error)

El servidor, ha encontrado un error que le impide realizar la solicitud

502 (Bad Gateway)

Se ha recibido un error por parte del Gateway

503 (Service Unavailable)

El servidor es incapaz de realizar la petición debido a una sobrecarga temporal o por mantenimiento del servidor

504 (Gateway Timeout)

El servidor, no ha tenido una respuesta a tiempo por parte del gateway.
