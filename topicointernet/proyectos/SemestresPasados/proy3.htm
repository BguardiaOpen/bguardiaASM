Desarrollo en Internet

Proyecto 3: Sistema de Información en Internet.

Equipos de 2-3 personas.

El objetivo del proyecto será proporcionar al usuario un conjunto de servicios a través de formas interactivas e información situada en una base de datos.

Entre las opciones de proyecto, que se sortearán o asignarán entre los equipos durante la clase, se tendrán las siguientes:

  • Sistema de información y reservaciones hoteleras.
  • Sistema de información y reservaciones de boletos aéreos.
  • Bolsa de Trabajo por Internet.
  • Anuncios clasificados por Internet.

Sistema de reservaciones hoteleras

Este sistema contará con los módulos que permitirán:

  • Buscar un cuarto disponible para cierta fecha o fechas consecutivas.
  • Reservarlo y ponerlo a nombre de un cliente.
  • Cancelar una reservación.

Para esto, la base de datos (puedes bajarla) se podrá accesar en el servidor mediante el DSN "hotel", y contará con las siguientes tablas y campos:

Tabla de "Clientes"
Nombre del campo Tipo Descripción
IDCliente Autonumérico Consecutivo automático que identifica de manera única al cliente; es llave.
Nombre Texto(50) Cadena que permite capturar el nombre completo del cliente.
Direccion Texto(50) Cadena que permite capturar la dirección del cliente.
RFC Texto(13) RFC para emitir factura
Compania Texto(50) Nombre de la compañía del cliente.
Telefonos Texto(50) Teléfonos del cliente.

 

Tabla de "Tarifas"
Nombre del campo Tipo Descripción
IDTarifa Autonumérico Identificador para las referencias de otras tablas a la tarifa; es llave.
NombreTarifa Texto(50) Nombre con el que se conoce y pide la tarifa; ej "Sencillo", "Doble", "Suite"...
ValorNoche Moneda Costo base por noche, en pesos
DescuentoAuto Moneda Máximo descuento autorizado, en pesos, que se aplica a discreción del que reserva.

 

Tabla de "Cuartos"
Nombre del campo Tipo Descripción
IDCuarto Numérico Número del cuarto que lo identifica en forma única dentro del hotel; es llave.
Capacidad Numérico Número de personas que pueden alojarse en dicho cuarto
Exterior Sí/No (Booleano) Si tiene vista hacia afuera o no
TipoTarifa Numérico Referencia a la tarifa que aplica por noche en este cuarto (ver tabla de Tarifas)

 

Tabla de "Reservaciones"
Nombre del campo Tipo Descripción
IDCuarto Numérico Número del cuarto que lo identifica en forma única dentro del hotel
Fecha Fecha/Hora Día para el que se aparta el cuarto. Junto con el anterior, forma la llave.
IDCliente Numérico Referencia a la tabla de clientes.
Tarifa Moneda Tarifa acordada (sale de tarifa base, y puede tener un valor entre esta y la misma menos el descuento)
FormaPago Texto(10) Indica si es en efectivo (EFEC), tarjeta (TARJ) o cheque (CHEQUE)
NumTarjeta Texto(50) En caso de que sea por tarjeta, aquí se registra el número de la misma.

Bolsa de Trabajo

Este sistema contará con los módulos que permitirán:

  • Registrarse cuando se busca trabajo, describiendo las habilidades principales y la experiencia laboral.
  • Como empresa, buscar un candidato que cumpla cierto criterio.

Para esto, la base de datos (puedes bajarla) se podrá accesar en el servidor mediante el DSN "bolsa", y contará con las siguientes tablas y campos:

Tabla de "Candidatos"
Nombre del campo Tipo Descripción
IDCandidato Autonumérico Consecutivo automático que identifica de manera única al candidato; es llave.
NombreCompleto Texto(50) Cadena que permite capturar el nombre del candidato
Direccion Texto(100) Generales del usuario
TelefonoCasa Texto(50) Generales del usuario
TelefonoOficina Texto(50) Generales del usuario
EMail Texto(50) Dirección de correo para contactarlo

 

Tabla de "ExperienciaLaboral"
Nombre del campo Tipo Descripción
ID Autonumérico Consecutivo automático que identifica de manera única al registro; es llave.
IDCandidato Numérico Referencia al candidato
NombreEmpresa Texto(50) Empresa en la que trabajó
FechaInicio Fecha/Hora Fecha en que inició labores en dicha empresa
FechaFin Fecha/Hora Fecha en que terminó labores en dicha empresa
Puesto Texto(50) Título del puesto que tenía
DescripcionPuesto Texto(50) Qué hacía en dicho puesto

 

Tabla de "Habilidades"
Nombre del campo Tipo Descripción
IDHabilidad Autonumérico Consecutivo automático que identifica de manera única la habilidad.
NombreHabilidad Texto(50) Nombre de la habilidad, ej "Inglés"
Tipo Numérico Indicador de formato: 1=Porcentaje (por ejemplo, Inglés es tipo porcentaje pues se maneja 0-100% de dominio); 2=Verdadero o Falso (habilidad que se tiene o no se tiene); 3=Años de experiencia.

 

Tabla de "HabilidadesCandidatos"
Nombre del campo Tipo Descripción
IDHabilidad Numérico Relación con la habilidad.
IDCandidato Numérico Relación con el candidato.
Valor Numérico Indica el valor de esa habilidad para ese candidato; por ejemplo si la habilidad fuera la 1 (inglés); dado que el tipo de la habilidad 1 es porcentaje, aquí estaría el valor del porcentaje. Si la habilidad fuera la 3 (disponibilidad para viajar) el valor sería 0 si no hay y 1 si la hay; etc.

Anuncios Clasificados

Este sistema contará con los módulos que permitirán:

  • Navegar por las secciones y sus anuncios, buscando información. Esto se presentará a todos los visitantes, estén registrados o no.
  • Registrarse como usuario para poner anuncios.
  • Agregar un anuncio en una sección existente. Esta opción requerirá estar registrado como usuario.
  • Quitar un anuncio, siempre y cuando sea el usuario que lo puso.

Para esto, la base de datos (puedes bajarla) se podrá accesar en el servidor mediante el DSN "clasificados", y contará con las siguientes tablas y campos:

Tabla de "Usuarios"
Nombre del campo Tipo Descripción
IDUsuario Autonumérico Consecutivo automático que identifica de manera única al usuario; es llave.
NombreUsuario Texto(50) Cadena que permite capturar el nombre que usará el usuario para registrarse en el sistema (alias)
ClaveUsuario Texto(50) Password usado para registrarse
NombreReal Texto(50) Nombre exigido al usuario para que el encargado del sistema sepa quién publica la información.
DireccionReal Texto(50) Generales del usuario
TelefonoReal Texto(50) Generales del usuario

 

Tabla de "Secciones"
Nombre del campo Tipo Descripción
IDSeccion Autonumérico Consecutivo automático que identifica de manera única a la sección; es llave.
NombreSección Texto(50) Nombre de la sección; ej. "Venta de automóviles", "Venta de equipo de cómputo", etc.

 

Tabla de "Anuncios"
Nombre del campo Tipo Descripción
IDAnuncio Autonumérico Consecutivo automático que sirve como llave
IDSeccion Numérico Referencia a la sección en la que va a salir.
TextoAnuncio Texto(50) Texto que sale en el anuncio.
Telefono Texto(8) Teléfono que acompaña al texto
FechaColocado Fecha/Hora Fecha y hora en la que se registró el anuncio en el sistema
FechaVigencia Fecha/Hora Fecha y hora en la que se retirará automáticamente el anuncio del sistema. Máxima vigencia de 2 semanas.
UsuarioPublica Numérico Referencia al usuario que publicó el anuncio.
